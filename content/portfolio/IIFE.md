#### Исходный код:
```
console.time('a');
((y = 2 ** 14 , x = Date.now()) => {
  while (Date.now() - x < y) ;
})();
console.timeEnd('a');
```
#### Объяснение
`console.time('a')` - Запускает таймер с меткой `a`

**IIFE** (Immediately Invoked Function Expression) - самовызывающаяся функция-выражение. Это функция, которая объявляется и тут же исполняется.

Внутри используется стрелочная функция =>, которая сама по себе не запускается. Чтобы она стала выражением, её оборачивают в скобки, а затем сразу вызывают, используя () в конце:
```
(( ) => { ... })();
```

В исходном коде мы задаем значения переменным: y = 2 ** 14 = 16384 мс (16 сек), x = Date.now() - текущее время. 
```
y = 2 ** 14 , x = Date.now()
```

Мы создаем "блокирующую задержку" - цикл, который длится y миллисекунд (≈ 16.384 секунды). Пока не пройдет это время (не завершится цикл), программа не будет выполняться дальше (в нашем случае, не зафиксируется время выполнения программы).
```
while (Date.now() - x < y) ;     // сравнение текущего времени с x
```

Когда цикл заканчивается, вызывается:
```
console.timeEnd('a');
```

Он выводит, сколько времени прошло между `console.time` и `console.timeEnd`. Получается, цикл затормаживает выполнение программы на 16348 мс, а затем выводит фактическое время выполнение (которое равно 16384 мс).

#### Альтернативный вариант без IIFE
```
console.time('a');

let y = 2 ** 14;
let x = Date.now();

while (Date.now() - x < y) {
}

console.timeEnd('a');
```

Здесь переменные задаются отдельно, затем запускается пустой цикл, который задерживает время выполнение всего кода на 16384 мс, что и фиксирует таймер. Результат выполнения этой версии задачи аналогичен первоначальному варианту.
